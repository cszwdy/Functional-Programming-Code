Using the Any type evades Swift's type system. However, trying to return 0 in the body of the noOp function defined using generics will cause a type error. Furthermore, any function that calls noOpAny does not know to which type the result must be cast. There are all kinds of possible runtime exceptions that may be raised as a result.
