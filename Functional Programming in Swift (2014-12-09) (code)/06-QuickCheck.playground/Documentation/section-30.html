Here we can see a good example of when QuickCheck can be very useful: it finds an edge case for us. If a size has exactly one negative component, our area function will return a negative number. When used as part of a CGRect, a CGSize can have negative values. When writing ordinary unit tests, it is easy to oversee this case, because sizes usually only have positive components. (The CGSize instances are omitted in this chapter, but can be found in the appendix.)
