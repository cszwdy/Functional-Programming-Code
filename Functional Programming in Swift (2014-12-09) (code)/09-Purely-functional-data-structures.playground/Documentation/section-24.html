In the text that follows, we will sometimes refer to the keys of type [T] as strings, and values of type T as characters. This is not very precise — as T can be instantiated with a type different than characters, and a string is not the same as [Character] — but we hope it does appeal to the intuition of tries storing a collection of strings.
