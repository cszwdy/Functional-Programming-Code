Here we can distinguish three cases:

  • The key is non-empty — in this case, we look up the subtrie corresponding to the first element of the key. If this also exists, we make a recursive call, looking up the tail of the key in this subtrie.
  • The key is non-empty, but the corresponding subtrie does not exist — in this case, we simply return false, as the key is not included in the trie.
  • The key is empty — in this case, we return isElem, the boolean indicating whether or not the string described by the current node is in the trie or not.
